# PRD — 캘린더 기반 포토 저널 (Photo Journal)

## 한 줄 정의
> 사진으로 하루를 저장하고, 달력으로 다시 보고, 화면 위에 띄우고, 월말엔 영상과 일기장으로 꺼내는 **개인 기억 아카이브**.

판단 기준: 새 기능을 넣을 때 "기록이 더 편한가"가 아니라 **"하루를 더 잘 남기고 나중에 더 잘 꺼내보게 만드는가"** 로 판단한다.

## 핵심 사용자
1. 사진은 많이 찍지만 기록은 오래 못 가는 사용자
2. 하루를 길게 쓰기보다 짧게 정리하고 싶은 사용자
3. 자기 기록을 예쁜 형태로 다시 보고 싶어 하는 사용자

## 제품이 해결할 일
1. 하루를 **90초 안에** 기록할 수 있어야 한다.
2. 기록이 쌓일수록 **달력이 예뻐져야** 한다.
3. 기록을 나중에 **영상·배경화면·일기 카드**로 다시 꺼낼 수 있어야 한다.
4. 사적인 데이터이므로 **비공개·잠금이 기본**이어야 한다.

## 제품 원칙
- 캘린더가 첫 화면이다. 앱을 열자마자 이번 달을 본다.
- 하루에는 대표 이미지가 반드시 있다. 텍스트보다 시각적 기억이 먼저 보인다.
- 글은 길이보다 **구조**가 중요하다 (오늘 한 줄 · 짧은 회고 · 감정 · 태그).
- 저장 즉시 결과물이 생긴다 (셀 썸네일 · 하루 카드).
- 회고는 자동으로 시작된다 (On This Day · 월말 리캡).
- 개인 기록은 기본적으로 잠겨 있다.

## MVP 범위 (이 웹 프로토타입)
포함 — **핵심 루프 전체**:
1. 월간 캘린더 홈
2. 하루 기록 작성 (대표사진 1 + 보조사진 ≤3, 오늘 한 줄, 짧은 회고, 감정, 태그, 메모)
3. 하루 카드 출력 (포스터 · 종이 일기장 · 미니멀 3종 템플릿, PNG export)
4. (덤) 월간 리캡 슬라이드 · On This Day 회고 진입점

제외 (MVP 비핵심): 소셜 피드, 팔로우, 협업 편집, 긴 블로그형 에디터, AI 채팅.

## 플랫폼 메모
기획안 권장은 iOS(SwiftUI+WidgetKit)+Supabase. 본 저장소는 **검증 가능한 웹 프로토타입**으로 UX·데이터모델·디자인 토큰을 먼저 확정하고, 이후 동일 모델을 네이티브로 이식한다. 기획안의 "web viewer/editor" 확장 단계와도 연결된다.

## 성공 지표 (행동 기준)
- 가입 후 첫 주 3일+ 기록 비율 / 작성 시작 대비 저장 완료율
- 저장 엔트리 중 대표사진 설정 비율 / 30일 재방문율 / 평균 기록 완료 시간
